CS-01

AWS Medallion Lakehouse with CDC

An end-to-end change-data-capture platform: every write to MySQL lands in an analytics-ready Redshift star schema, incrementally, with full history.

MySQL RDS→ DMS cdc→ S3 bronze→ Glue + PySpark silver→ Redshift gold→ QuickSight

the problem

Batch reloads don’t scale and lose history. The platform needed row-level change tracking from an operational MySQL database into a warehouse, with slowly changing dimensions preserved, and it had to stay cheap to scan.

decisions that mattered

  • Hash-based SCD Type-2 — SHA-256 row fingerprints detect real changes, so Redshift merges only touch actual deltas instead of diffing every column.
  • DMS batch-apply mode — killed the micro-file problem in S3 that was quietly destroying downstream Glue performance.
  • Partition pruning + predicate pushdown — Glue jobs scan only the ingestion-date partitions they need; Parquet + Snappy cut storage and scan cost.
  • Step Functions ordering — dimensions load before facts, so referential integrity survives an asynchronous pipeline.
  • ROW_NUMBER() dedup — window logic guarantees exactly one active record per business key during merges.

outcome

Incremental, restartable ETL from operational database to BI dashboards with full change history, the architecture pattern I now reach for first when someone says “we need analytics on our transactional data.”

CS-02

Enterprise Procurement Data Platform

Production system at Genpact for GE Vernova: eight ERP source systems, one trustworthy view of the procurement lifecycle for renewable-energy operations.

SAP · Oracle · BAAN · DOOSAN +4 more→ Databricks bronze→ harmonize silver→ PSOT / CSOT gold→ Tableau · Power BI

the problem

Purchase orders, shipments, receipts and payables for wind-turbine equipment live in eight different ERP systems that disagree with each other. Reporting teams needed one source of truth, refreshed on SLA, reconciled to the cent.

decisions that mattered

  • YAML-driven DAG factory — Airflow workflows are declared as config (schedule, retries, timezone, tags), not copy-pasted Python. New pipelines ship in minutes.
  • Filter pushdown, aggressively — moved cancel-flag, import-flag and authorization logic upstream, cutting query latency 50% and saving major annual manual effort.
  • PSOT/CSOT gold modeling — separate “primary” and “consumer” sources of truth so BI teams get curated views without touching core datasets.
  • Reconciliation as a discipline — traced currency-code mismatches, NULL propagation and PO-distribution inconsistencies across source systems by following lineage layer by layer.

outcome

Data-quality scores up 15%, query latency halved, fewer data incidents and a monitoring stack (Airflow, Step Functions, Tableau, Grafana) that surfaces SLA breaches before stakeholders do.
